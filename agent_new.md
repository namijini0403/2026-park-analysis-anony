# 비식별 제출 앱 작업 지침

## 역할

이 저장소는 기존 식별 가능 앱을 대체하거나 수정하는 곳이 아니다. 기존 앱의 화면 구조, 정책 흐름, 상호작용을 최대한 보존하되, 제출용 공개 환경에서 학교와 주변 시설이 재식별되지 않도록 별도로 구현하는 비식별 앱 저장소다.

## 절대 원칙

- 기존 저장소 `2026-park-analysis`의 파일을 수정하지 않는다.
- 이 저장소는 기존 저장소의 fork나 branch가 아니라 독립 저장소로 유지한다.
- 원본 학교명, 원본 학교 ID, 실제 위도/경도, 주소, 실제 공원명, 실제 아파트 단지명, 실제 재개발 구역명, 실제 지도 타일을 공개 산출물에 넣지 않는다.
- 원본 CSV, GeoJSON, XLSX, 실명 매핑 파일은 GitHub에 올리지 않는다.
- 제출 앱이 읽는 데이터는 `public/data/*.json`의 비식별 산출물뿐이어야 한다.
- 변환 스크립트는 로컬 원본을 읽을 수 있지만, 출력 JSON에는 식별자가 남으면 안 된다.
- 기능을 단순화해야 할 때도 기존 앱의 정책적 의미를 보존한다.

## 목표 화면 구조

기존 앱과 같은 사용자 흐름을 살린다.

1. 첫 화면
   - 앱 요약, 분석 취지, Human-in-the-loop 메시지
   - 전체 앱으로 들어가는 시작 버튼

2. 지도 화면
   - 실제 지도 API를 쓰지 않는다.
   - 줌인/선택 흐름은 유지한다.
   - 학교별 500m 반경, 도보 500m 범위, 공원, 후보지, 단절요소의 상대 위치를 도식으로 보여준다.
   - 학교와 시설은 비식별 코드 또는 `공원 A`, `후보지 A`, `단절요소 A`처럼 표시한다.

3. 학교 상세 리포트
   - 기존 상세 리포트의 지표 구조를 최대한 유지한다.
   - 현재 격차, 미래 수요, 접근 마찰, 유사학교 비교를 분리해 보여준다.
   - 유사학교도 비식별 코드로 표시한다.

4. 후보지 시뮬레이션
   - 기존 후보지 추천 흐름과 필터/가중치 개념을 유지한다.
   - 후보지는 실제 grid ID 대신 `후보지 A/B/C`로 표시한다.
   - 실제 좌표 대신 학교 중심 상대거리와 경로 부담만 사용한다.

5. 전체 통계
   - 기존 전체 통계 화면의 구 단위 비교 흐름을 유지한다.
   - 학교명 Top 목록은 모두 비식별 코드로 표시한다.

## 구현 태도

- 한 번에 전체를 갈아엎지 말고 단계별로 작업한다.
- 각 단계는 “기존 구조 대응표”, “식별자 제거 확인”, “화면 확인”을 통과해야 다음 단계로 넘어간다.
- 기존 앱의 디자인을 참고하되, 원본 코드를 무분별하게 복사하지 않는다. 복사할 경우 식별자 흐름을 먼저 제거한다.
- 공개 가능한 산출물과 로컬 전용 입력물을 명확히 분리한다.
- `npm run check:safety`를 작업 중 자주 실행한다.

## 커밋 전 확인

```bash
npm run build:data
npm run build
npm run check:safety
git status --short
git ls-files
```

커밋 대상에 원본 데이터 파일, 실명 매핑 파일, 원본 좌표 파일, 빌드 캐시, 로그가 보이면 중단한다.
